## 快速开始

### 环境要求

- Node.js 16+ 
- SQL Server 2016+
- npm 或 yarn

### 1. 安装依赖

```bash
cd server
npm install
```

### 2. 配置数据库

编辑 `server/config/index.js`：

```javascript
module.exports = {
  database: {
    user: 'your_username',
    password: 'your_password',
    server: 'localhost',
    database: 'M-A_OP_ODB'
  },
  jwtSecret: 'your-secret-key',
  port: 3005
}
```

### 3. 初始化数据库

运行数据库初始化脚本：

```bash
cd server/database
# 执行必要的数据库初始化脚本
```

### 4. 启动服务器

```bash
cd server
npm start
```

服务器将在 http://localhost:3005 启动

### 5. 配置 Web 服务器

#### 使用 Nginx（推荐）

配置文件 `/etc/nginx/sites-available/ma-system`：

```nginx
server {
    listen 80;
    server_name your-domain.com;
    
    root /path/to/V1.0/frontend;
    index index.html;
    
    location / {
        try_files $uri $uri/ /index.html;
    }
    
    location /api/ {
        proxy_pass http://localhost:3005;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}
```

启用配置：

```bash
ln -s /etc/nginx/sites-available/ma-system /etc/nginx/sites-enabled/
nginx -t
systemctl restart nginx
```

#### 使用 Apache

配置文件 `/etc/apache2/sites-available/ma-system.conf`：

```apache
<VirtualHost *:80>
    ServerName your-domain.com
    DocumentRoot /path/to/V1.0/frontend
    
    <Directory /path/to/V1.0/frontend>
        Options -Indexes +FollowSymLinks
        AllowOverride All
        Require all granted
        
        RewriteEngine On
        RewriteCond %{REQUEST_FILENAME} !-f
        RewriteCond %{REQUEST_FILENAME} !-d
        RewriteRule ^.*$ /index.html [L,QSA]
    </Directory>
    
    ProxyPass /api http://localhost:3005
    ProxyPassReverse /api http://localhost:3005
</VirtualHost>
```

## 常见问题

### Q: 端口 3005 被占用

**解决方案 1**：停止占用端口的进程
```bash
# Windows
netstat -ano | findstr :3005
taskkill /F /PID <进程 ID>

# Linux
lsof -i :3005
kill -9 <进程 ID>
```

**解决方案 2**：修改端口号
编辑 `server/server.js`，将端口改为其他值（如 3006）

### Q: 数据库连接失败

检查项：
1. SQL Server 服务是否运行
2. 数据库 `M-A_OP_ODB` 是否存在
3. 用户名密码是否正确
4. 防火墙是否允许连接

### Q: 前端页面空白

1. 打开浏览器开发者工具（F12）
2. 查看 Console 错误信息
3. 检查 Network 标签页 API 请求是否成功
4. 确认 API 地址配置正确

### Q: 登录失败

1. 确认后端服务器正常运行
2. 检查数据库用户表是否有数据
3. 查看服务器日志错误信息
