1.2 架构核心强制规则
主数据管控规则
客户、物料、标准供应商、员工、部门、标准 BOM、标准化产品，仅由 MDM 主数据平台统一创建、编码与分发，各业务系统禁止自建标准主数据，仅通过 API 与消息订阅同步使用。适配非标制造场景：非标工序、定制参数、工装夹具、流程自定义字段开放白名单，允许各模块自主维护，不纳入 MDM 强管控。主数据变更全量推送事件，下游自动同步，保障基础数据统一。
事件驱动分层规则
全域核心业务状态变更，必须按统一 JSON Schema 发布事件至消息总线，事件强制包含：ID、时间戳、源系统、业务类型、实体数据、操作人。跨系统协作仅允许事件订阅 + 网关 API 调用，禁止跨库直连、跨库联表。流程引擎内部节点流转、审批状态变更为引擎闭环，按需选择性发布事件，避免消息冗余；所有跨系统业务动作由事件订阅触发。
底座优先级规则
平台基础底座强制优先级：流程引擎 → 消息总线 → MDM 主数据 → API 网关，全业务模块必须基于底座能力构建。
入口统一规则
普通员工仅通过统一门户操作，聚合待办、审批、单据、报表，禁止直连独立业务后台。流程配置、MDM 管理、运维类专业后台，仅对管理员授权开放，严格权限隔离。
数据闭环规则
业务数据经消息总线汇入数据中台 ODS/DWD 分层，统一输出报表与看板。生产异常、设备故障、售后报修、质量不合格等事件，自动同步研发、质量体系，联动知识库更新，形成制造改善闭环。
1.3 模块边界联动规则
保留 CRM、SRM、MES、WMS、质量、研发等原有模块职责与事件输出规范。跨部门、跨系统串联审批、驳回回流、会签或签，统一由全局流程引擎集中编排；各业务模块仅负责自有数据管理，禁止跨模块硬编码耦合。
1.4 开发强制规范
接口遵循 RESTful，统一接入 API 网关做鉴权、限流、监控。事件 Schema 固定规范，仅允许新增扩展字段，禁止随意修改核心结构。数据库单库隔离，禁止跨库关联查询，跨实体统计统一由数据中台处理。前端统一登录、权限、UI 风格；全服务集成日志、链路追踪、健康监控。本地开发固定前端 3000、后端 3005；测试、生产环境取消固定端口，采用网关路由 + 服务发现。
1.5 实施阶段强制顺序
底座搭建：消息总线部署 + 制造业流程引擎开发（节点画布、版本、启停能力）
主数据治理：MDM 落地、编码规范、主数据事件同步配置
业务改造：各模块接入流程引擎、完成事件改造、主数据对接
数据建设：数据中台分层建模、统一报表与看板开发
能力聚合：API 网关、统一门户、单点登录、权限聚合
闭环落地：知识库迭代、售后 / 质量 / 研发业务闭环上线
1.6 流程引擎强制规则
流程引擎为平台核心基础组件，所有业务单据必须绑定流程定义，无流程单据禁止流转。内置设备制造业专属节点：发起人、部门主管审批、角色审批、抄送、会签、或签、条件分支、驳回回流、超时控制、自动处理节点。流程全生命周期能力必达：新建、编辑、复制、启用 / 停用、冻结、版本管理、回退、按模块 / 单据 / 部门绑定生效。各模块支持流程自定义配置、自定义业务字段，制造特殊参数独立存储，不侵入标准主数据。全流程操作留痕、审批日志可追溯、支持导出归档，满足制造业质检追溯与内审合规要求。
